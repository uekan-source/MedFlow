MedFlow

薬局における欠品対応を一元管理する業務支援Webアプリ

サービス概要

薬局で発生する薬剤の欠品対応を一元管理するWebサービスです。
欠品の発生から入荷後の患者連絡までの対応状況を可視化します。
属人化しがちな業務を仕組みで支え、連絡漏れや引き継ぎミスを防ぎます。

このサービスへの思い・作りたい理由

私は薬局薬剤師として働く中で、欠品時対応が現場に大きな負担を与えていると感じてきました。

欠品が発生した場合、患者への連絡、入荷後の再連絡、対応状況の共有など、
複数の業務が同時並行で発生しますが、それらは紙や口頭で管理されることが多く、
「誰が・どこまで対応したのか」が分かりにくい状況でした。

実際に、欠品していた薬が入荷し調剤準備が完了していたにもかかわらず、
患者への連絡を失念してしまい、患者からお叱りを受けた経験があります。

この経験から、
個人の注意力に依存した業務フローには限界があり、
仕組みとして対応状況を管理・可視化する必要があると強く感じました。

本サービスは、欠品対応に関わる情報を一元管理し、
医療従事者が安心して患者対応に集中できる環境を作ることを目的としています。

ユーザー層について
薬局薬剤師・薬局スタッフ

欠品対応の中心となる業務を担っており、
日常業務の中で最も負担を感じやすい層であるためです。

薬局管理者

業務の属人化や引き継ぎミスを防ぎ、
店舗全体の業務状況を把握する必要がある立場であるためです。

サービスの利用イメージ

薬剤師が欠品情報を登録すると、
欠品状況・患者連絡の有無・対応ステータスが一覧で確認できます。

入荷後は「調剤準備完了」「患者連絡済み」などのステータスを更新することで、
対応漏れを防ぎ、チーム全体で進捗を共有できます。

その結果、連絡忘れや引き継ぎミスが減り、
安心して患者対応に集中できる環境を実現します。

サービスの差別化ポイント・推しポイント

・一般的なタスク管理ツールではなく、薬局の欠品対応フローに特化
・欠品 → 入荷 → 調剤準備 → 患者連絡という実務の流れをそのままステータス化
・現役薬剤師の実体験をもとに設計

現場で「無理なく使える」ことを重視しています。
